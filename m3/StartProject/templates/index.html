 Help on the module webpage

  The webpage has 2 main sections: the main menu un the upper left corner and the center area.
The menu on the upper left corner of the page contains 3 buttons: “Add Item”, “Save” and “Reload”.

  Main menu

  The Add Item button

  Upon selection, it will prompt a new window that will ask the user to enter the required data for a new relation. The new window will ask the user to fill the following fields:
From concept: the base concept
Relation: the relation between the two concepts
To concept: the target concept
Value: what kind of relation it is (unidirectional or adirectional)
Text: phrases where this relation is identified
All fields are mandatory.

After completing all fields with the right type of data, the user can either save the newly entered relation by pressing the “Save" button or cancel the operation by pressing “Cancel”. If the new relation imputed by the user leads to the formation of a cycle between the concepts.

  The Save button

  This button allows the user to save all the modifications made. The modifications can be: the addition of new relations, the deletion of existing relations, the removal of relations.

  The Reload button

  Its effect is similar to a rollback. It goes back to the state before any modification performed by the user. After the user clicks on this button, it will be asked to confirm the action and acknowledge that all the work that was not save will be lost in the process.

  The Help button

  It opens a popup page in which the purpose of the site is presented and the actions allowed.

  Center area

  The center region of the page has the search bar. Below the search bar there is a table with the relevant data to be displayed. The table has 7 columns:
From: first concept
Relation: name of the relation between the two concepts
To: second concept
Type: simple graphical representation of the relation - which can be either unidirectional or adirectional
Value: corresponding number for the relation
Text: propositions where the relation is identified
Action: two buttons
An Edit button used to modify data about the entry and a Remove button that deletes the relation altogether.

Below the table it's the webpage pagination.

  The Check History button

  At the bottom of the page it's the “Check History” button. This leads the user to a new page, similar to the center region of the main page.
It has the search bar at the top and a table below it. In the table there are all the modifications performed on the ontology, with the date on which it was done, the content that was changed, and the options to restore or remove the modifications.

Date: The date at which the modification was performed. It is displayed in the format: yyyy-mm-dd hh:mm:ss.
Differences: section of data edited in the JSON file (modified, added or removed)
Actions: two buttons; restore modification or remove modification from history.
Below the table we have the pagination.
